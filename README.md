회원가입, 로그인, 로그아웃, 계정 탈퇴 등 계정 기능 전체 구현

커뮤니티 게시글/댓글/좋아요 기능 구현

마이프로필 페이지 (계정·프로필 수정, 좋아요 글 조회)

Kakao Map API 연동을 통한 주변 식당 조회

메인 페이지 기능 병합 및 통합 작업 참여

🛠️ 기술 스택

Frontend: Vue.js 3

Backend: Django REST Framework

State Management: Pinia

API: Kakao Map API

Collaboration: GitHub, Notion

🚧 핵심 기술적 도전
1. 로그인 이후 사용자 흐름 설계

자동 로그인 상태에서도 프로필 입력 여부에 따라 페이지 분기

인증 + 프로필 상태를 함께 판단하는 구조 설계

FE/BE가 동일한 사용자 상태 기준을 갖도록 API 및 상태 관리 구현

2. 메인 페이지 컴포넌트 구조 개선

부모–자식 컴포넌트 과도한 중첩으로 인한 유지보수 어려움

기능 단위 컴포넌트 분리 및 전역 상태 관리로 구조 단순화

📈 배운 점

인증은 단순 로그인 구현이 아닌 사용자 상태 흐름 설계 문제임을 이해

상태 관리, 라우팅, 컴포넌트 구조가 유기적으로 연결되어야 함을 경험

기능 구현보다 구조와 책임 분리가 프로젝트 완성도를 좌우한다는 점을 학습

FE와 BE를 모두 구현하며 요청부터 화면 반영까지의 전체 흐름을 이해
